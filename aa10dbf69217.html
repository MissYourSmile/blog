把系统从 deepin 重装成 ubuntu

发表于 2019-11-25 更新于 2021-10-23 分类于 环境搭建
记录从 deepin 系统迁移到 ubuntu 系统的过程，保留了 home 分区的内容。

准备工作

记下分区挂载点

在安装之前，要先记下各个分区的挂载点，以便后续安装过程中进行挂载。

从图中命令的结果可以得到以下几条信息

  • 计算机内当前有 2 个存储设备，/dev/sda 和 /dev/sdb
  • /dev/sda 是电脑内置的固态硬盘，分了四个分区，可通过 df 命令查看挂载点。特别要记清根目录和 home 目录的挂载位置。
分区名称 类型 挂载点
/dev/sda1 EFI System /boot/efi
/dev/sda2 Linux swap
/dev/sda3 Linux filesystem /
/dev/sda4 Linux filesystem /home`
  • /dev/sdb 是插入电脑的 U 盘，用来作新系统启动盘。

制作U盘

使用下面的命令来制作启动盘

  • xxxx.iso 处填写镜像文件的完整路径
  • of 参数一定要填 U 盘对应的路径名，注意：不带数字，不是 /dev/sdb1
  • 这里会等待比较长的时间，且终端不会有任何显示，不要误以为卡了。

开始安装

启动盘制作完成后，关机，开机按对应的键进入 BIOS，选择 U 盘启动，开始 ubuntu 的安装，以下是需要注意的一些细节。

  • 点击安装前最好断网，因为 ubuntu 会在安装时下载更新，下载的速度较慢，可以等系统装完之后更改了镜像源用国内源进行更新。
  • 在选择安装在何处时，选择最下面的自定义，在自定义界面，按照之前记下的挂载点进行挂载。根挂载点所在分区格式化，home 只挂载不做格式化操作。

ps： 这一步忘了留下珍贵的图像资料，将就着看下文字吧。

设置完毕点击下一步，会提示将格式化两个分区，一个是 efi， 一个是 / 所在分区

完成

等待安装完成，一个崭新的 ubuntu 又出现了，哈哈哈。
